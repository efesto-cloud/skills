# Entity: Invoice

## Purpose
An Invoice represents a request for payment issued to a customer for goods or services rendered. It is the central aggregate of the billing domain and tracks the full lifecycle from draft creation through payment or cancellation.

## Fields

| Field         | Type              | Rules / Constraints                                          |
|---------------|-------------------|--------------------------------------------------------------|
| id            | string            | UUID v4, immutable after creation                            |
| invoiceNumber | string            | Human-readable identifier (e.g. INV-2024-0042), immutable    |
| customerId    | string            | UUID of the customer, immutable after creation               |
| amount        | Money             | Value object (amount: number, currency: string); amount > 0  |
| lineItems     | LineItem[]        | At least one item required; immutable after issuance         |
| status        | InvoiceStatus     | See status transitions below                                 |
| issuedAt      | Date \| null      | Set when transitioning to `issued`; null in `draft`          |
| createdAt     | Date              | Set on creation, never updated                               |
| updatedAt     | Date              | Updated on every mutation                                    |

## Status transitions

```
[draft] → [issued] → [paid]
                   ↘ [cancelled]
[draft] → [cancelled]
```

Rules:
- WHEN status is `paid` THEN no further transitions are allowed
- WHEN status is `cancelled` THEN no further transitions are allowed
- WHEN transitioning from `draft` to `issued` THEN `issuedAt` must be set and `lineItems` must not be empty
- WHEN transitioning from `issued` to `paid` THEN the transition is triggered by an external payment event (recorded via `MarkInvoicePaidUseCase`)

## Invariants

- An invoice must always have at least one line item
- `amount` must equal the sum of all line item amounts
- `customerId` is immutable — an invoice cannot be reassigned to a different customer
- WHEN status is `issued` or `paid` THEN `lineItems` are frozen and cannot be modified
- WHEN status is `draft` THEN `issuedAt` must be null

## DTOs

| DTO Name                  | Purpose                          | Fields included                                      |
|---------------------------|----------------------------------|------------------------------------------------------|
| `InvoiceDto`              | Full read representation         | All fields                                           |
| `CreateInvoiceInputDto`   | Create a new draft invoice       | `customerId`, `lineItems`, `currency`                |

No `UpdateInvoiceInputDto` — invoices are immutable after creation except through explicit use cases (`IssueInvoiceUseCase`, `MarkInvoicePaidUseCase`, `CancelInvoiceUseCase`).

## Domain errors originating from this entity

| Error name                      | When it is thrown                                       |
|---------------------------------|---------------------------------------------------------|
| `InvoiceNotFoundError`          | Invoice with given id does not exist                    |
| `InvalidInvoiceStatusError`     | Attempted an illegal status transition                  |
| `InvoiceAlreadyIssuedError`     | Attempted to modify line items after issuance           |
| `EmptyInvoiceLineItemsError`    | Attempted to issue an invoice with no line items        |
