# Spec Templates

Templates for creating new entity and use case specifications.

## Templates

### [TEMPLATE-MINIMAL.md](./TEMPLATE-MINIMAL.md)
**Best for**: Quick idea capture  
**Time**: 5 minutes  
**Use when**: You have a rough idea and want to capture it fast

Minimal bullet-point format. Great for brainstorming and initial planning.

---

### [TEMPLATE-CONVERSATIONAL.md](./TEMPLATE-CONVERSATIONAL.md) ⭐ Recommended
**Best for**: Standard specs  
**Time**: 15-30 minutes  
**Use when**: Planning a feature before implementation

Natural language, sentence-based format. Recommended for most specs.

---

### [TEMPLATE.md](./TEMPLATE.md)
**Best for**: Comprehensive documentation  
**Time**: 30-60 minutes  
**Use when**: Complex features needing full structure

More rigid, table-based format. Ensures you cover everything.

---

## How to Use

1. **Copy the template** that fits your needs
2. **Rename** to `YourEntityName.md` or `YourUseCaseName.md`
3. **Fill in** the sections (delete what you don't need)
4. **Save** in appropriate directory:
   - Entities → `../entities/YourEntity.md`
   - Use cases → `../usecases/YourUseCase.md`
   - Rough drafts → `../drafts/idea.md`
5. **Update index** files to include your new spec
6. **Link** to related specs using markdown links

## Tips

- **Start minimal**, add detail as you go
- **Don't overthink** - you can always refine later
- **Be conversational** - write like you're explaining to a teammate
- **Use markdown links** to reference other specs
- **Mark status**: 🔵 Draft → 🟡 Review → 🟢 Approved → ⚪ Implemented
